# Командное задание: Мини‑проект

## Задача
1. Соберите команду 3–5 человек, заведите репозиторий `projects/<team-name>/` или отдельный репозиторий в организации.
2. Сформулируйте цель (SMART) и гипотезу MVP.
3. Спланируйте 2 спринта по 1–2 недели, настройте Kanban и роли.
4. По итогам: демо, релиз (tag), README с инструкцией запуска.

## Сдача
- PR с папкой проекта или ссылкой на проект в организации, + `report.md`.
