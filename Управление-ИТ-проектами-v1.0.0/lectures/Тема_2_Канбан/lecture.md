# Тема 2. Канбан в GitHub

## 1. Канбан и поток работ
- Принципы: визуализация, WIP-лимиты, управление потоком, непрерывное улучшение.
- Польза для учебного курса: прозрачность статусов, снижение перегрузки, быстрые фидбэки.

## 2. GitHub Projects (новая версия)
- Views: Board, Table, Roadmap.
- Поля: Status, Assignee, Milestone, Sprint (custom), Score.
- Автоматизация: правила переноса карточек по меткам и событиям PR/Issue.

## 3. Настройка доски курса
- Колонки: `To Do → In Progress → Submitted → Graded`.
- Метки: `lab01`, `lab02`, `to-review`, `changes-requested`, `accepted`.
- Автосвязь: PR ↔ Issue, автозакрытие по `Fixes #ID`.

## 4. WIP-лимиты и баланс нагрузки
- Пример: не более 2 открытых задач на студента.
- Спринты: 1–2 недели, демо на семинарах.

## 5. Метрики
- Lead Time, Cycle Time.
- Диаграмма кумулятивного потока (вручную в табличке / внешнем инструменте).
