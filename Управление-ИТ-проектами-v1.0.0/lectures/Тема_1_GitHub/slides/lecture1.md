# Тема 1
## GitHub — основной инструмент управления ИТ‑проектами

---

## Зачем GitHub в учебном курсе?
- Централизация материалов и заданий
- История изменений и прозрачность
- Практики, близкие к индустрии

---

## Базовые сущности
- repo / branch / commit
- Pull Request (код‑ревью)
- Issues, Labels, Projects (Kanban)

---

## Workflow сдачи
1. Публикация задания
2. Fork и выполнение
3. Pull Request + автотесты
4. Комментарии и доработки
5. Merge и оценка

---

## Канбан в GitHub
- Колонки: To Do → In Progress → Submitted → Graded
- Метки и автоматизация

---

## Лучшие практики
- Маленькие PR
- Понятные сообщения коммитов
- Чёткие критерии оценивания
