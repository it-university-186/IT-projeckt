# Тема 1. GitHub — основной инструмент управления ИТ‑проектами

## 1. Зачем GitHub в учебной дисциплине
- Центр управления: код, задания, обсуждения, доски.
- Транспарентность и история изменений (коммиты, PR).
- Приближено к индустриальным практикам.

## 2. Базовые сущности
- Репозиторий, ветка, commit, Pull Request (PR), Issue.
- Labels, Milestones, Projects (Kanban).

## 3. Рабочий процесс (workflow) курса
1) Преподаватель публикует задание в `assignments/` и создаёт Issue.  
2) Студент делает fork, выполняет задание в своей ветке/форке, открывает PR.  
3) Запускается автопроверка (Actions).  
4) Преподаватель комментирует, просит доработки или принимает PR.  
5) Оценка фиксируется в PR/Issue и в `assignments/.../evaluation.md`.

## 4. Канбан-доска курса
- Колонки: `To Do` → `In Progress` → `Submitted` → `Graded`.
- Автоматизация через метки и правила.

## 5. Лучшие практики
- Осмысленные сообщения коммитов.
- Маленькие PR по одному логическому изменению.
- Чёткие критерии в `evaluation.md`.
